1. Executive Summary
The current Landing Builder uses GrapeJS as its page editing engine. While functional, it presents significant limitations in UX, performance, element library richness, customization depth, responsive behavior, and — most critically — it lacks any native funnel logic that allows users to build real conversion journeys.
This PRD proposes a complete rebuild of the Builder module, transforming it from a single-page editor into a full Funnel Builder system inspired by ClickFunnels. The new module will move us from building isolated landing pages to building end-to-end conversion paths: landing → opt-in → sales → checkout → upsell → downsell → thank you → membership.
1.1 Why Now
GrapeJS has reached its ceiling for our product requirements.
User feedback consistently flags UX complexity as the top friction point.
Competitors (ClickFunnels, Systeme.io, Leadpages) are pulling ahead with funnel-native logic.
Our market (MENA + global SMBs) demands Arabic/RTL support, faster performance, and sales-flow native tooling.
Laravel Blade stack allows server-side rendering advantages that competitors (mostly SPA-based) cannot match.

11. Forms & Lead Capture
11.1 Overview
Forms are first-class citizens in the funnel system. Every form submission creates a Lead, which is automatically linked to a Contact in the CRM and can trigger email sequences or webhook events.
11.2 Form Builder
Drag-and-drop field builder within the form element
Field types: text, email, phone, number, textarea, checkbox, radio, dropdown, date, file upload, hidden field, consent
Per-field validation: required, regex, min/max, custom error messages
Conditional fields (show field B only if field A = value X)
Multi-step forms (wizard mode) with progress indicator
Pre-fill from URL parameters (utm_*, email, name)
Honeypot + reCAPTCHA v3 anti-spam
Phone input with country code selector (MENA countries prioritized)
Arabic-ready placeholders and labels
11.3 Submit Actions
Multiple actions can run in parallel: e.g., Go to next step + Trigger sequence + Webhook.

20.1 Timeline Overview
Total to v2.0 stable release: ~20 weeks (5 months). Full roadmap through v2.2: ~9–10 months.
20.2 Migration Strategy
Existing v1 users must not be forced to rebuild. Migration path:
Automated conversion tool: reads GrapeJS JSON → maps to new element structure
Coverage target: 80% of common patterns auto-migrate perfectly
For unsupported patterns: visual warning + side-by-side manual rebuild helper
v1 remains accessible for 6 months post-v2 launch (read-only after 3 months)
Migration guide + 1-on-1 help for paying customers during transition
20.3 Release Strategy
Closed beta — 200 invited users (Phase 1.5)
Public beta — opt-in for existing users (after 4 weeks)
GA — available to all new signups (after 2 more weeks)
v1 deprecation announcement — 3 months before shutdown
