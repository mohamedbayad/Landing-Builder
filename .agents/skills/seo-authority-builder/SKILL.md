# SEO Authority Builder

Analyzes content for E-E-A-T (Experience, Expertise, Authoritativeness, Trustworthiness) signals and suggests improvements to build authority. Essential for YMYL (Your Money Your Life) topics.

---

## When to use this skill

- When auditing content for E-E-A-T signal strength
- When building trust and authority for a website or brand
- When working on YMYL topics (health, finance, legal, safety)
- When creating author bios, about pages, or trust elements
- When implementing topical authority through content clusters

## How to use it

### Understanding E-E-A-T

E-E-A-T is **not a direct ranking factor** — it's a framework search engines use to evaluate content quality.

| Dimension | What It Represents | Common Signals |
|---|---|---|
| **Experience** | First-hand, real-world involvement | Original examples, demonstrations, lived experience |
| **Expertise** | Subject-matter competence | Credentials, depth, accuracy |
| **Authoritativeness** | Recognition by others | Mentions, citations, links |
| **Trustworthiness** | Reliability and safety | HTTPS, transparency, accuracy |

### Step 1 — Audit Current Signals

Score each E-E-A-T dimension (1-10):

**Experience signals:** First-hand indicators, case studies, original research, behind-the-scenes content, process documentation.

**Expertise signals:** Author credentials, technical depth, industry terminology, comprehensive topic coverage, expert quotes.

**Authority signals:** External links from authoritative sites, brand mentions, industry recognition, published research.

**Trust signals:** Contact information, privacy policy, SSL, reviews/testimonials, editorial guidelines.

### Step 2 — Create Enhancement Plan

```
Current Score: X/10
Target Score: Y/10

Priority Actions:
1. Add detailed author bios with credentials
2. Include case studies showing experience
3. Add trust badges and certifications
4. Create topic cluster around [subject]
5. Implement Organization schema
```

### Step 3 — Implement Trust Optimization

- About page enhancement with team bios
- Editorial policy / fact-checking process
- Contact accessibility (email, phone, address)
- Social proof integration (reviews, logos, testimonials)

## Examples

### Example 1 — Health Product Landing Page (YMYL)

**Input:** Blood pressure monitor landing page  
**Audit:**
```
Experience: 3/10 — No customer stories showing real use
Expertise: 4/10 — No medical professional endorsement
Authority: 2/10 — No press mentions or certifications
Trust: 5/10 — Has guarantee but no contact info
```
**Recommendations:**
- Add testimonial from a nurse or healthcare worker
- Display FDA clearance badge
- Add clinical accuracy statement with citation
- Include contact information and return policy

### Example 2 — E-commerce Product Page

**Input:** Tire inflator product page  
**Audit:**
```
Experience: 7/10 — Has use case photos and customer reviews
Expertise: 6/10 — Spec comparisons present
Authority: 3/10 — No press or industry mentions
Trust: 8/10 — Clear guarantee, verified reviews, HTTPS
```
**Recommendations:**
- Add "As Seen In" press bar with logo mentions
- Include expert review quote from automotive publication
- Add Organization schema markup

---

## Constraints

- **Do NOT** fabricate E-E-A-T signals — they must be based on real credentials and evidence
- **Do NOT** add generic trust badges that don't link to verifiable sources
- **Do NOT** skip trust signals for YMYL topics — they are critical for ranking
- **ALWAYS** prioritize Trustworthiness — it's the most important E-E-A-T dimension
- **ALWAYS** include author attribution for content in sensitive categories
- **ALWAYS** implement Organization or Person schema for entity recognition
